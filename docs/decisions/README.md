# Architecture Decision Records (ADRs)

ADRs document significant architectural decisions. They are created for **Tier S** changes (schema, security, infrastructure).

## When to create an ADR
- Database schema changes
- Auth/security behavior changes
- New external dependency or API integration
- Deployment topology changes
- Breaking API contract changes

## Template
Copy `docs/decisions/000-template.md` and increment the number.
